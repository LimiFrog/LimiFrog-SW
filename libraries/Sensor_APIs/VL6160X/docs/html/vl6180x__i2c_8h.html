#define I2C_BUFFER_CONFIG   1

Configure device register I2C access.

  • 0 : one GLOBAL buffer
    Use one global buffer of MAX_I2C_XFER_SIZE byte in data space
    This solution is not multi-device compliant nor multi-thread cpu safe
    It can be the best option for small 8/16 bit MCU without stack and limited ram (STM8s, 80C51 ...)
  • 1 : ON_STACK/local
    Use local variable (on stack) buffer
    This solution is multi-thread with use of i2c resource lock or mutex see VL6180x_GetI2CAccess()
  • 2 : User defined
    Per device potentially dynamic allocated. Requires VL6180x_GetI2cBuffer() to be implemented.
